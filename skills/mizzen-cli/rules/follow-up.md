# 追问深度策略

访谈中 AI 主持人的追问行为由 `--follow-up` 参数控制。

## 如何选择追问级别

| 题目类型 | 推荐级别 | 理由 |
|----------|----------|------|
| 甄别题 | `none` | 快速筛选，不需要展开 |
| 选择题 | `none` | 选完即过 |
| 暖场题 | `light` | 建立信任，但不需要深挖 |
| 收尾题 | `light` | 简单收束 |
| 核心探索题 | `heavy` | 深入理解动机和体验 |
| 关键决策题 | `heavy` | 挖掘决策过程 |

## none — 不追问

**设置**：`--follow-up none`

**行为**：
- 收到任何可理解的回答后，立即进入下一题
- 即使回答很短（一个词）、不完整或敷衍，也不追问
- 唯一例外：完全无法理解的内容（乱码、噪音）

**适用场景**：
- 甄别题（screening）
- 纯事实性问题（"你用的是什么手机？"）
- 选择题

## light — 轻度追问

**设置**：`--follow-up light`

**行为**：
- 仅在回答太简短（不到 2 句话）时追问
- **最多追问 1 次**
- 目的：让受访者把话说完整，不是深挖

**触发条件**：
- 敷衍回答（"还行"、"挺好的"）
- 回答模糊不清
- 回答与问题不太对应

**适用场景**：
- 暖场题
- 收尾题
- 事实性但需要稍微展开的问题

## heavy — 深度追问

**设置**：`--follow-up heavy`

**行为**：
- 主动追问 1-3 次，深入探索话题
- 目的：理解"为什么"、发现有趣的细节、挑战不一致的说法
- 澄清模糊回答的追问不算在次数内

**追问方向**：
- 为什么？（动机）
- 具体怎么做的？（过程）
- 当时感觉怎么样？（情感）
- 能举个例子吗？（具象化）

**适用场景**：
- 核心探索问题
- 用户体验和决策过程
- 需要深入了解的关键话题

## auto — 自动判断

**设置**：`--follow-up auto`

**行为**：AI 主持人根据受访者的回答自动决定是否追问以及追问深度。一般不推荐使用——明确指定 none/light/heavy 能让访谈节奏更可控。

**适用场景**：不确定该用哪个级别时的兜底选项。

## 追问的通用规则

不管什么级别，追问都必须遵守：

1. **所有追问必须是开放式的**
   - ✅ "为什么选择了这款？"
   - ❌ "是因为价格还是品质？"（二选一）
   - ❌ "是不是因为朋友推荐？"（引导性）

2. **引用受访者的具体内容**（主动倾听）
   - ✅ "你提到了'操作太麻烦'，能具体说说是哪个环节让你觉得麻烦吗？"
   - ❌ "能再多说一点吗？"（太泛）

3. **一次只追问一个方向**，不要同时问两件事

4. **问 why / how / what**，不问 yes/no

## CLI 示例

```bash
# 甄别题：不追问
mizzen outline question add <slug> <section-id> \
  --text "你多久网购一次？" \
  --type multiple_choice \
  --options "每天,每周,每月,很少" \
  --follow-up none

# 暖场题：轻度追问
mizzen outline question add <slug> <section-id> \
  --text "你平时喜欢什么休闲活动？" \
  --type open_ended \
  --follow-up light

# 核心题：深度追问
mizzen outline question add <slug> <section-id> \
  --text "你上次退掉一个已购买的产品是什么情况？为什么要退？" \
  --type open_ended \
  --follow-up heavy
```
