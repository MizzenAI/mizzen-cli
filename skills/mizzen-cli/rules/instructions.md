# addInstructions 编写指南

`addInstructions`（访谈指引）是写给 AI 主持人看的指令，控制条件跳转和追问方向。

## 什么时候用

- 需要根据前面的回答跳过或进入特定题目
- 需要引导 AI 主持人往特定方向追问
- 需要根据回答内容做分支

**谨慎使用**，只在真正需要时添加。大部分题目不需要 addInstructions。

## 四种条件类型

### 1. 单条件跳转

```
如果用户选择"否"，跳到 Q8。
```

### 2. 分支跳转（多路径汇合）

```
如果选"选项A"：问 Q6 → Q7 → Q8，然后跳到 Q15。
如果选"选项B"：问 Q9 → Q10 → Q11，然后跳到 Q15。
如果选"选项C"：问 Q12 → Q13 → Q14，然后跳到 Q15。
```

### 3. 多条件组合

```
如果用户在 Q3 回答了"是"并且在 Q5 选了"每天"：问 Q6 → Q7 → Q8，然后继续到 Q9。
否则：跳过 Q6、Q7、Q8，直接到 Q9。
```

### 4. 开放式内容条件

```
如果用户的回答表明高频使用（出现"每天"、"天天"、"经常"等词语）：问 Q5 → Q6 → Q7，然后继续到 Q8。
如果是低频或中频：跳过 Q5、Q6、Q7，直接到 Q8。
```

## 编写规范

- 用题号（Q1、Q2）指定跳转目标
- 用箭头（→）表示分支内的顺序
- 每个分支末尾用"然后跳到 Qxx"标明汇合点
- 分支内的题目不需要单独写 addInstructions（按顺序执行）
- 筛选受访者用甄别题（screening），不要用条件跳转

## 追问引导用法

除了条件跳转，addInstructions 还可以写追问引导：

```
如果用户提到了价格因素，追问：具体在什么价格范围内你觉得合理？
如果用户提到了竞品，追问：和你现在用的产品相比，最大的区别是什么？
```

AI 主持人会把这些引导作为追问的主要方向，优先于自由发挥。

## CLI 示例

```bash
# 条件跳转
mizzen-cli outline question add <slug> <section-id> \
  --text "你是否有过网购退货的经历？" \
  --type multiple_choice \
  --options "有,没有" \
  --follow-up none \
  --instructions "如果用户选择'没有'，跳到 Q6。"

# 追问引导
mizzen-cli outline question add <slug> <section-id> \
  --text "你选择这款产品的主要原因是什么？" \
  --type open_ended \
  --follow-up heavy \
  --instructions "如果用户提到朋友推荐，追问具体是怎么推荐的、推荐了什么信息。如果提到广告，追问是在哪里看到的广告。"
```
