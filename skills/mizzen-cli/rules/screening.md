# 甄别题设计

## 什么时候需要甄别

- **从样本库招募**：可选，仅在研究者明确要求时添加
- **自行招募**：通常不需要（研究者直接招募目标人群）
- 不要主动创建甄别题，除非用户要求

## 基本规则

- 最多 5 道题（越少越好）
- **只用选择题**，不用开放题
- **追问设为 none**——甄别环节不追问
- 创建题目时用 `--options` 的 `+/-` 前缀标记选项状态：`+选项` = 通过，`-选项` = 筛除
- 示例：`--options "+全职上班,+兼职,-学生,-退休,-待业"`

## 筛选逻辑

- 选了任何 `reject` 选项 → 筛除（即使同时选了 `approve`）
- 没选 `reject` + 至少选了一个 `approve` → 通过
- 只选了未标记状态的选项 → 筛除

## 三大设计原则

### 1. 合并原则（Consolidation）

目标人群合并成一个宽泛选项，不要拆得太细。

❌ 错误：前端开发、后端开发、全栈开发、移动开发（太明显在找程序员）
✅ 正确：软件工程师（一个选项，混在其他职业中）

### 2. 分散原则（Diversification）

把少量 `approve` 选项藏在大量不相关的 `reject` 选项中。

**示例**：筛选软件工程师
```
- 软件工程师 (approve)
- 医疗保健从业者 (reject)
- 教师 (reject)
- 零售管理人员 (reject)
- 建筑工人 (reject)
- 会计师 (reject)
```

大部分选项是 reject，目标选项不突出。

### 3. 陷阱选项（Trap Options）

加入虚假但逼真的选项，抓不诚实的受访者。

**要求**：
- 微妙且专业（不能明显是假的）
- 符合真实选项的命名风格
- 正常人不会选（因为不存在），但看起来合理

**示例**：
- 支付平台：在 支付宝、微信支付 旁边加 "汇智付"（不存在）
- 航空公司：在 东方航空、南方航空 旁边加 "华中航空"（不存在）
- 游戏：在 王者荣耀、原神 旁边加 "幻域传说"（不存在）

**适用**：多选的产品/服务使用、活动、技能类问题
**不适用**：单选题、人口统计、地理、宽泛职业分类

## 访谈标题伪装

如果甄别题涉及品牌，访谈标题不要暴露。

❌ "PlayStation 用户调研"
✅ "游戏偏好调研"

## CLI 示例

```bash
# 创建甄别板块
mizzen outline section add <slug> -t "基本信息" --type screening

# 添加甄别题（用 +/- 前缀标记 approve/reject）
mizzen outline question add <slug> <section-id> \
  --text "你目前的职业是？" \
  --type multiple_choice \
  --options "+软件工程师,-医疗保健从业者,-教师,-零售管理人员,-建筑工人,-会计师" \
  --follow-up none
```

`+` = approve（通过），`-` = reject（筛除），无前缀 = neutral。
